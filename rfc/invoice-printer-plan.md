# 发票打印助手（invoice-printer）设计方案

> 状态：已定稿（Approved），2026-07-01 修订：统计仅输出到控制台，不再写入 PDF。
> 日期：2026-06-30
> 作者：wangjian（经由 opencode 协同）

## 1. 背景与目标

报销时需把大量电子发票（火车票、打车票、住宿票）打印成纸质件。手工逐张排版效率低、浪费纸张，且金额需要人工汇总易错。

目标：一个命令行 Rust 工具，输入一个发票目录，输出一份排版好的 A4 PDF，并对金额分类汇总。

## 2. 需求

1. 指定本次需报销的发票目录（递归扫描）。
2. 读取所有发票（PDF）。
3. 火车票：适配 A4 竖版，每页 8 张，4 行 × 2 列，不足 8 张自上而下顺序排列。
4. 普通发票：适配 A4 竖版，每页 2 张，2 行 × 1 列，自上而下排列。
5. 对称分布，方便裁剪为 A5 大小。
6. `test` 目录为测试发票。
7. 控制台分类输出统计金额：火车票、打车费、住宿费、合计。
8. 金额同时给出中文大写（控制台）。
9. 所有 A4 页面为竖版；**每页画一条水平中心虚线**，沿虚线裁切即得上下两片 A5。

## 3. 现状分析（test 目录共 22 张 PDF）

| 类型 | 文件 | 页面尺寸(pt) | 检测关键词 | 金额样例 |
|---|---|---|---|---|
| 火车票 Train | f00001–f00016 | 595×397(横版) | `铁路电子客票` | 票价￥642 / ￥620 |
| 打车 Taxi | f00017–f00019 | 595×409 | `客运服务费`/`滴滴`/`交通运输服务` | 价税合计小写 ¥129.50 |
| 住宿 Hotel | f00020–f00022 | 595×397 | `住宿服务`/`酒店` | 价税合计小写 ¥189.00 |

**金额提取风险：** 住宿发票文字抽取会跨行断裂（`（小写）¥189.`↵`00`），需用 `pdftotext -layout` 或 pdfium 文字抽取，定位"小写"后折叠空白再匹配。

**环境：** cargo 1.96；已装 poppler（pdftotext/pdftoppm）；未装 ghostscript、pdfium 库。

## 4. 技术决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| PDF 引擎 | `pdfium-render` | 矢量排版质量最高；单一依赖同时负责文字抽取+排版+合成；API 支持 Form XObject 嵌入页面 |
| 输出结构 | 单个合并 PDF | 打印最省事 |
| 版面标注 | 不在发票页加标注 | 保持原貌 |
| 中心虚线 | 仅水平中心线(H/2) | 裁成 2 片 A5；版面干净 |
| 纸张方向 | 全部 A4 竖版 | 需求明确 |

## 5. 依赖

`pdfium-render`(PDF)、`clap`(CLI)、`anyhow`(错误)、`regex`(金额)、`walkdir`(扫描)、`comfy-table`(控制台表)。
运行时：`lib/libpdfium.dylib`（由 `scripts/fetch-pdfium.sh` 从 bblanchon/pdfium-binaries 下载，按路径加载）。

## 6. 模块结构

```
invoice-printer/
  Cargo.toml
  scripts/fetch-pdfium.sh   # 下载 libpdfium(darwin-arm64) → lib/
  lib/libpdfium.dylib       # 运行时按路径加载
  fonts/NotoSansSC-Regular.otf
  src/
    main.rs           编排
    cli.rs            Args{ dir(默认./test), out(默认./output.pdf) }
    scanner.rs        walkdir 收集排序 *.pdf
    invoice.rs        Invoice{ path, kind, amount }; Kind{Train,Taxi,Hotel,Other}
    extractor.rs      pdfium 抽文字 → 分类 → 正则金额
    chinese_yuan.rs   f64 → 发票大写（含单元测试）
    layout.rs         A4 竖版 n-up 几何 + 统计表 A5 矩形 + 中心线坐标
    composer.rs       源页 Form XObject 缩放居中；每页画水平中心虚线
    report.rs         控制台表（含大写列）
```

## 7. 排版几何（A4 竖版 W=595.28 H=841.89 pt）

- **火车票** 2 列×4 行=8 格：列中心 W/4, 3W/4；行中心 H/8, 3H/8, 5H/8, 7H/8；源页(595×397)等比缩放居中于格内（留 ~10pt padding）。中心线 H/2 落在第 2、3 行之间 → 每片 A5 含 4 张(2×2)。
- **普通发票** 1 列×2 行=2 格：行中心 H/4, 3H/4；每格 = A5(595×420pt)；中心线 H/2 落在两行之间 → 每片 A5 含 1 张。
- **中心虚线**（每页）：水平线 (0,H/2)→(W,H/2)，灰色、线宽 ~0.5pt、dash 模式 [3,3]。若 pdfium dash API 受限则用短线段模拟。

## 8. 分类与金额提取

**分类（优先级从高到低）：**
1. 含 `铁路电子客票` → Train
2. 含 `住宿` 或 销方 `酒店` → Hotel
3. 含 `客运服务费` / `滴滴` / `交通运输服务` → Taxi
4. 其余 → Other（按普通发票排版，独立计数）

**金额正则：**
- 火车票：`票价[：:]\s*[￥¥](\d+\.\d+)`
- 普通发票：定位"小写"后取子串，折叠所有空白（含换行）后匹配首个 `[￥¥](\d+\.\d+)`

**大写转换规则：** 整数部分按万亿分组（零合并），接"圆"；角→"X角"，分→"X分"；无分则末尾加"整"。
示例：11051.50 → 壹万壹仟零伍拾壹圆伍角整。

## 11. 容错

- 缺 `libpdfium.dylib`：输出下载提示（运行 `scripts/fetch-pdfium.sh`）。
- 金额解析失败：告警并跳过该张的统计（仍排版）。
- 非 PDF / 损坏文件：跳过并告警。
- 某类为空：正常处理，金额记 0。
- 目录不存在：报错退出。

## 12. 执行步骤

1. `cargo init invoice-printer`，配置 Cargo.toml 依赖。
2. 下 libpdfium 到 `lib/`，放 Noto Sans SC 到 `fonts/`。
3. 按 §6 实现 8 个模块 + 单元测试（chinese_yuan、extractor、layout）。
4. `cargo run -- --dir test --out output.pdf`，核对页数、中心虚线、金额大写。
5. `cargo test` 通过。
